## 重大 bug 修复：BEV y 轴索引左右镜像
- 问题：`PA_gen_v2/osz_source.py` 和 `OSZ/modules/ray_casting.py` 中 BEV 的 y 轴索引公式是 `j = int((y - y_min) / res)`，与 `imshow(extent=[y_max, y_min, ...], origin='lower')` 的显示方向相反，导致 ego 左侧（+y）被存到数组右侧。
- 后果：`is_in_osz` 查的是左右镜像后的 OSZ 单元格；车辆明明在道路上，程序误判为在 OSZ 中；画出来红 PA 框不在 OSZ 区域内。
- 修复：
  - `PA_gen_v2/osz_source.py`：`bev_xy_to_ij` 改为 `j = int((y_max - y) / res)`；`ij_to_bev_xy` 同步修正；磁盘缓存 hash 加 `_yflip_fixed` 后缀并删除旧缓存。
  - `OSZ/modules/ray_casting.py`：体素 `ys` 改为从 `y_max` 到 `y_min`；`build_bev_occ_from_pointcloud` 的 `yi` 改为 `(y_max - y) / res`；ego ray casting 的 `ego_yi` 同步。
  - `OSZ/modules/drivable_filter.py`：PIL 栅格化的 `row` 改为 `(y_max - y) * scale_y`，保证可行驶区域 mask 与 OSZ mask 同向。
  - `OSZ/visualize/bev_viz.py`：`plot_gt_osz` 的 `in_osz` 检查改为 `yi = int((y_max - cy) / res)`。
  - `common/bev_config.py` 与 `PA_gen_v2/osz_geometry_legacy.py`：`bev_coords_to_pixel` / `pixel_to_bev_coords` 同步修正。
- 验证：随机 ego 坐标 → BEV 索引 → imshow 显示位置，y 值已正确对应（左侧=+y）。
- 影响：所有旧 OSZ 缓存失效，已挖掘的 ghost events 需要重新跑 miner。
