## visualize_events.py 升级为交互式 Event Browser
- 将 `PA_gen_v2/visualize_events.py` 从纯离线 PNG 导出升级为支持交互浏览的质检工具。
- 新增 `EventBrowser` 类 + `launch_browser()`，2x3 布局：
  `[t-4][t-3][t-2] / [t-1][t][info]`，键盘 n/p/r/q (next/prev/redraw/quit)。
- **关键设计：每帧用各自时刻的 ego 坐标系**（ego 恒在原点），
  显示该帧自己的 PA-relevant OSZ——这样能直观看到 OSZ 随时间演变、
  车辆何时从 OSZ 出来、OSZ 是否移动、ego 是否转弯。
  不同于原 `visualize_event()`（把所有 lookback 帧变换到 frame-t ego 系）。
- lookback 帧车辆位置：直接标注优先，缺失则 `trajectory.locate_at_time`
  插值（与 `ghost_vehicle_miner` 判定逻辑一致，QA 视图与 miner 不会矛盾）。
  frame t 用存储的 `emerge_bev_xy` 精确绘制（专用 `_draw_frame_own_ego_emerged`）。
- 右侧 info panel：event index/label、scene name、token、lookback 逐帧
  彩色判定表、每帧 OSZ 覆盖率、OSZ 重叠数、阅读提示、键盘说明。
- 去掉模块顶部 `matplotlib.use('Agg')`（破坏 plt.show()）；新增
  `_ensure_gui_backend()` 探测 WebAgg/TkAgg/QtAgg。保留原离线导出能力。
- main 增加 `--browse` / `--label_filter` / `--start_idx`；原 `make_event_grid` /
  `print_event_stats` / `visualize_event` 保留不变，向后兼容。

## 用户本地运行（Windows, E:\ 路径）卡住问题
- 现象：命令行打印到 `[osz_source] RayCaster3D grid...` 后"没反应"。
- 根因：**GUI 模式 `plt.show()` 是阻塞调用，终端卡住属正常**；且 Tk 原生窗口其实已弹出
  （用户 managed python 有 tkinter，tornado 缺 → 不是 WebAgg 问题），只是被挡住没注意。
- 修复/改进：
  1. 后端优先级调整：TkAgg/QtAgg 原生窗口排前，WebAgg 降到最后（仅远程无显示时用）。
  2. `EventBrowser.launch()` 增加后端提示（WebAgg 时打印 http://localhost:8988/）。
  3. 新增显式 `--headless` 开关 → 强制终端+PNG 模式。
  4. **headless 改为每个事件存独立 PNG**：`output/browser/event_0000.png, event_0001.png...`
     （原只覆盖单张 `_browser_current.png`，不便查看）。命令行 n/p/j/k/数字/q 翻页。

## 新增 analyze_distances.py 统计脚本
- 分析 762 个 positive 事件的 emergence 距离分布，无需 nuScenes-devkit，
  直接读 JSON（需 numpy + matplotlib）。
- 输出：
  1. `output/distance_analysis.png` — 2×2 图（直方图、每箱计数、Unknown 比例、
     OSZ overlap）。
  2. `output/distance_analysis.csv` — 每箱统计表。
  3. 终端报告 — 累计留存率、数据驱动的阈值建议。
- 关键统计结果：
  - 距离 min/max/mean/median = 3.6/66.3/31.5/31.1 m。
  - Unknown 比例在 ≤30m 仅 ~5%，30-40m 约 11-13%，之后无持续悬崖。
  - OSZ overlap 均值 ≤30m 约 2.6-4.0，30-40m 约 2.3-2.4。
- 数据驱动的阈值结论：**没有明显的 35m/40m 质量悬崖**，退化是渐进的。
  35-40m 是较实用的平衡点（保留 62-76% 的样本，strong% 仍约 75%）。
  收紧到 25-30m 会损失约 54-69% 的样本，收益有限。
- 阈值逻辑说明：从峰值质量区间（10-15m）向外扫描，找持续 ≥2 个箱退化
  （unknown≥15% 或 osz_mean<2 或 strong%<60%），避免误把 0-5m 近距离
  低 OSZ 重叠判为崩塌点。
- 运行：`python PA_gen_v2/analyze_distances.py`
  （可选：`--bin 10` 改箱宽，`--out <png>` 改输出路径）

## 性能优化：cast_osz_2d 向量化 + OSZ 磁盘缓存
- **根因定位**：`--web` 模式生成 80 个事件 × 5 帧 = 400 个 sample 的 OSZ，
  每个 sample 的 `cast_osz_2d`（2D 阴影射线投射）用**纯 Python 双重 for 循环**
  跑 ~2500 万次迭代（n_angles=12566 × max_steps=2000），单 sample 约 25 秒。
  80 个事件需 3-5 小时 → 用户看到只有 RayCaster3D 刷屏、output/web 无文件。
- **修复 1：向量化 `cast_osz_2d`**（`OSZ/modules/ray_casting.py`）
  - 把 12,566 条射线用 numpy 同时推进，外层循环从 2500 万次降到 2000 次。
  - 正确性：20 次随机小网格测试 bit-for-bit 一致。
  - 性能：500×500 网格 25s → 0.5s，**提速 50 倍**。
- **修复 2：OSZ 磁盘缓存**（`PA_gen_v2/osz_source.py`）
  - `get_pa_relevant_osz_for_sample()` 新增三层缓存：内存 → 磁盘(.npz) → 计算。
  - 磁盘缓存在 `output/osz_cache/{config_hash}/` 下，按 sample_token 存 npz。
  - config_hash 含 BEV_RANGE/RESOLUTION/Z 参数，改配置自动失效。
  - 第一次运行算完存盘；第二次运行秒出（直接 load npz）。
- **修复 3**：`--web_max` 默认从 80 降到 20（首次运行约 5 分钟可完成）。
- 三个文件均通过 py_compile 校验。
- 运行：`python visualize_events.py --dataroot <nusc> --web`
  （首次 ~5-15s/event；第二次运行 cached → 秒出）

  所有 BEV 真值及对应地图，才能直观看出筛选对不对。

  2. **全场景 annotation BEV 框**：`_get_scene_annotations_ego()` 获取
     当前 sample 所有标注物体，用 `_bev_box_corners_ego()` 计算旋转矩形
     角点，`_draw_annotation_boxes()` 绘制：
     - 其他车辆 = 青色半透明框
     - 行人 = 黄色小点
     - 被追踪的幽灵车辆 = verdict 颜色框 + 白色朝向箭头
  3. **HD 地图车道线**：`_draw_map_overlay()` 从 NuScenesMap 加载车道
     边界多边形，变换到 ego 系，画成细蓝线（zorder=1 底层）。
     地图按 location 缓存（`_map_cache`），用 node-based 方式访问
     polygon（不依赖 shapely）。map_expansion 不可用时静默跳过。
  4. **info panel 图例**：新增颜色图例说明（红=OSZ, 灰=障碍, 绿=路面,
     青=其他车辆, 黄=行人, 蓝=车道线, ★=追踪车辆）。
- **坐标变换**：annotation 全局坐标 → ego 系用已有的 `_global_to_ego()`；
  朝向用 `ego_q.inverse * ann_q` 的 yaw_pitch_roll[0]（相对 ego 的偏航角）。
  画图时 swap (ego-x, ego-y) → (ego-y, ego-x) 配合 matplotlib 轴约定。
- **调用点更新**：`_draw_frame_own_ego` 和 `_draw_frame_own_ego_emerged`
  均加入 map overlay + annotation boxes；后者新增 `instance_token` 参数
  以高亮被追踪车辆。两处调用点（GUI + Headless _render）同步更新。
- **验证**：BEV box 角点计算 5 个方向测试（0/π/2/π/-π/2）全部通过；
  pyquaternion 相对朝向提取验证（0.7 rad 精确匹配）。py_compile 通过。
- **zorder 分层**：地图车道线(1) < imshow overlay < 其他车辆框(3)
  < 追踪车辆框(5) + 箭头(6) < marker/文字。
