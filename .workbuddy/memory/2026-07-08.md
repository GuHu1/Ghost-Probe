# 2026-07-08

## visualize_events.py 升级为交互式 Event Browser
- 将 `PA_gen_v2/visualize_events.py` 从纯离线 PNG 导出升级为支持交互浏览的质检工具。
- 新增 `EventBrowser` 类 + `launch_browser()`，2x3 布局：
  `[t-4][t-3][t-2] / [t-1][t][info]`，键盘 n/p/r/q (next/prev/redraw/quit)。
- **关键设计：每帧用各自时刻的 ego 坐标系**（ego 恒在原点），
  显示该帧自己的 PA-relevant OSZ——这样能直观看到 OSZ 随时间演变、
  车辆何时从 OSZ 出来、OSZ 是否移动、ego 是否转弯。
  不同于原 `visualize_event()`（把所有 lookback 帧变换到 frame-t ego 系）。
- lookback 帧车辆位置：直接标注优先，缺失则 `trajectory.locate_at_time`
  插值（与 `ghost_vehicle_miner` 判定逻辑一致，QA 视图与 miner 不会矛盾）。
  frame t 用存储的 `emerge_bev_xy` 精确绘制（专用 `_draw_frame_own_ego_emerged`）。
- 右侧 info panel：event index/label、scene name、token、lookback 逐帧
  彩色判定表、每帧 OSZ 覆盖率、OSZ 重叠数、阅读提示、键盘说明。
- 去掉模块顶部 `matplotlib.use('Agg')`（破坏 plt.show()）；新增
  `_ensure_gui_backend()` 探测 TkAgg/QtAgg。保留原离线导出能力（savefig 不依赖 Agg）。
- main 增加 `--browse` / `--label_filter` / `--start_idx`；原 `make_event_grid` /
  `print_event_stats` / `visualize_event` 保留不变，向后兼容。
- 校验：py_compile + AST 通过；接口名（osz_source / ghost_vehicle_miner /
  trajectory）经 grep 确认存在。未做真实 import（环境无 venv/nuscenes-devkit）。
- 运行：`python visualize_events.py --dataroot <nusc> --browse`
