## visualize_events.py 升级为交互式 Event Browser
- 将 `PA_gen_v2/visualize_events.py` 从纯离线 PNG 导出升级为支持交互浏览的质检工具。
- 新增 `EventBrowser` 类 + `launch_browser()`，2x3 布局：
  `[t-4][t-3][t-2] / [t-1][t][info]`，键盘 n/p/r/q (next/prev/redraw/quit)。
- **关键设计：每帧用各自时刻的 ego 坐标系**（ego 恒在原点），
  显示该帧自己的 PA-relevant OSZ——这样能直观看到 OSZ 随时间演变、
  车辆何时从 OSZ 出来、OSZ 是否移动、ego 是否转弯。
  不同于原 `visualize_event()`（把所有 lookback 帧变换到 frame-t ego 系）。
- lookback 帧车辆位置：直接标注优先，缺失则 `trajectory.locate_at_time`
  插值（与 `ghost_vehicle_miner` 判定逻辑一致，QA 视图与 miner 不会矛盾）。
  frame t 用存储的 `emerge_bev_xy` 精确绘制（专用 `_draw_frame_own_ego_emerged`）。
- 右侧 info panel：event index/label、scene name、token、lookback 逐帧
  彩色判定表、每帧 OSZ 覆盖率、OSZ 重叠数、阅读提示、键盘说明。
- 去掉模块顶部 `matplotlib.use('Agg')`（破坏 plt.show()）；新增
  `_ensure_gui_backend()` 探测 WebAgg/TkAgg/QtAgg。保留原离线导出能力。
- main 增加 `--browse` / `--label_filter` / `--start_idx`；原 `make_event_grid` /
  `print_event_stats` / `visualize_event` 保留不变，向后兼容。
- 校验：py_compile + AST 通过；接口名经 grep 确认存在。

## 无头模式修复（远程服务器适配）
- 用户远程 Linux 服务器无 tkinter/Qt → RuntimeError: No interactive backend。
- **修复**（同一文件内）：
  1. `_ensure_gui_backend()` 新增 `WebAgg` 首选候选（需 tornado，浏览器交互）
  2. 新增 `HeadlessEventBrowser` 类：Agg + 终端 stdin（n/p/j/k/q/数字/r），零 GUI 依赖
  3. `launch_browser()` 自动降级：GUI 失败 → 打印 ⚠ → 进入 headless 模式
- 输出 PNG 到 `output/_browser_current.png`（每次覆盖）；用户可用 image viewer 查看。
- **Bug 修复**：HeadlessEventBrowser._render() 中 `trajectory.build_instance_trajectories`
  → 改为 `build_instance_trajectories`（顶部是 `from trajectory import ...` 函数级导入，
  模块名 `trajectory` 未导入，导致 NameError）。
- **静态审计又发现 3 处 headless 类里的同类隐患（已修复）**：
  1. `LOOKBACK_K` 未定义 → 从 ghost_vehicle_miner 导入 `LOOKBACK_FRAMES` 并替换
     （否则首次渲染 `range(LOOKBACK_K)` 即 NameError）。
  2. `_draw_info_panel(...)` 参数顺序错位（frames/coverages/label_filter 三参位置错）→
     对齐签名 `(ax,nusc,event,idx,total,label_filter,coverages,frames)`，否则 info panel 内
     `zip(frames, coverages)` 收到 int 而 TypeError。
  3. `axes[2, 2]` 越界（subplots(2,3) 仅 0/1 两行）→ 改 `axes[1, 2]`（右下），否则 IndexError。
  - GUI 版 EventBrowser 对应调用均正确，仅 headless 版（我新写的）有这些笔误。
